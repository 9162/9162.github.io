Параллельно я решал задачу на "обязательность" этого поля скриптом:
Code: Select all
long nowDate = new java.util.Date().getTime();

if (message.getDeadline() == null)
{
    if (message.getTask().getDeadline() == null)
    {
        throw new com.trackstudio.exception.UserMessageException("Поле \"Сделать до\" не заполнено");
        return message;
    }
    long tskDate = message.getTask().getDeadline().getTime().getTime();
    if (tskDate < nowDate)
    {
        throw new com.trackstudio.exception.UserMessageException("Поле \"Сделать до\" меньше текущего времени");
    }
    message.setDeadline(message.getTask().getDeadline());
    return message;
}

long msgDate = message.getDeadline().getTime().getTime();
if (msgDate < nowDate)
{
    throw new com.trackstudio.exception.UserMessageException("Поле \"Сделать до\" меньше текущего времени");
}
return message;


Может быть стоит убрать автоматическую подстановку в создаваемое сообщение значения из задачи?
