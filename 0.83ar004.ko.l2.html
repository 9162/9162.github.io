[TwitStar] New Version (2.0) Released!

TwitStar 2.0이 출시되었습니다. (현재 앱스토어에서 무료로 다운로드 가능)

앱스토어로 바로 가기

자세한 내용은 TwitStar 홈페이지 (http://dearto20.com/twitstar)에서도 확인하실 수 있습니다.

지난 버전에 대한 여러분들의 많은 충고와 조언을 바탕으로 상당한 기능 개선 및 추가가 이루어졌습니다.

(아래 설명에서 사용된 이미지들은 TwitStar의 컬러 테마들 중 Charcoal Gray가 적용된 모습입니다. 기본값은 Caribbean Blue입니다. 아래에서 자세히 설명하겠지만 화면 하단 탭 바 중 가장 우측에 위치한 More로 진입하면 컬러 테마를 설정하는 것이 가능합니다.)

추가된 내용들에 대한 간단한 설명입니다.

1. 멀티 계정 지원 기능이 추가되었습니다.

<그림 1. 멀티 계정 설정>

기본 타임라인 보기 화면에서 좌측 상단의 Accounts 버튼을 누르면 바로 계정 선택화면으로 이동합니다.
이 곳에서 계정의 추가 / 삭제가 가능합니다. (아직 계정 생성 기능은 지원하지 않습니다. 계정 생성은 트위터 홈페이지에서 하실 수 있습니다.)

각 계정 별 내용들은 모두 따로 저장 / 관리되며, 특히 컬러 테마 설정 값 또한 각 계정 별로 각각 저장됩니다.
즉, 각각의 계정이 서로 다른 컬러 테마를 사용하는 것이 가능합니다. (예> 계정 1은 Snow White, 계정 2는 Charcoal Gray 사용)

2. 각 사용자의 상세 보기 화면이 기존 버전에 비해 대폭 개선되었습니다.

<그림 2. 사용자 정보 상세 보기>

기본적으로 사용자 프로필, 사용자 타임라인 (직접 작성한 트윗들), 팔로잉 목록, 팔로워 목록을 손쉽게 확인할 수 있습니다. (각 항목 별 수치 및 상세 내용 보기 지원)

또한 해당 사용자의 리스트 사용 정보도 상세하게 확인할 수 있습니다. (이 부분은 아래 리스트 관련 항목에서 보다 자세하게 설명)

그리고 스크롤을 조금 아래로 내려보면 현재 이 사용자와 나 (TwitStar 사용자)와의 팔로우 관계가 양방향으로 보여집니다. 즉, 이 사용자가 나를 팔로우 하고 있는지 여부, 내가 이 사용자를 팔로우 하고 있는지 여부를 한 눈에 확인할 수 있습니다. (물론, 이 사용자에 대한 팔로우 / 언팔로우는 간단하게 원 터치로 설정할 수 있습니다.)

3. 리스트 관리 기능이 추가되었습니다.

<그림 3. 사용자 별 리스트>

위 (2. 사용자 정보 상세 보기) 의 각 사용자 별 상세 보기 화면에서 Lists 를 누르시면 해당 사용자에 대한 상세한 내용을 확인할 수 있습니다.

그림에서 보는 것과 같이, 사용자 리스트는 크게 1. 사용자가 직접 생성한 리스트, 2. 사용자가 팔로우하고 있는 리스트, 3. 사용자를 팔로우하고 있는 리스트의 세 가지로 나누어집니다.

<그림 4. 리스트 타임 라인>

특정 리스트를 선택해서 들어가면 위 그림과 같이 하단에 Follow List / Unfollow List 가 보여지게 됩니다. 이 버튼을 이용해서 해당 리스트의 팔로우 / 언팔로우를 간단하게 설정할 수 있습니다.

4. 상, 하로 스크롤 이동 시 자동으로 리프레쉬가 이뤄지도록 하였습니다.

<그림 5. 리프레쉬>

위쪽 끝까지 스크롤을 하면 자동으로 최신 글을 불러오는 기능이 실행되며, 아래쪽 끝까지 스크롤을 하게 되면 자동으로 이어서 지난 글들을 불러오게 됩니다.

5. 자신이 쓴 글 (트윗)을 삭제할 수 있는 기능이 추가되었습니다.

<그림 6. 트윗 삭제하기>

기본 타임라인 보기 화면 또는 특정 트윗 상세 보기 화면에서 해당 트윗이 자신이 작성한 트윗일 경우, RE, RT, DM 아이콘 대신 휴지통 아이콘이 표시됩니다. 이를 통해 자신이 작성한 트윗을 쉽게 구분하는 것이 가능하며, 해당 아이콘을 터치하여 손쉽게 트윗을 삭제할 수 있습니다.

기존 앱들의 경우, 한 뎁스 진입해서 삭제하는 것이 일반적이나, TwitStar에서는 상세 보기 화면에 진입하지 않고도 손쉽게 자신이 작성한 트윗을 삭제하는 것이 가능합니다.

6. 새 글 시작 지점 표시 기능이 추가되었습니다.

<그림 7. 새 글 시작 지점 표시>

TwitStar 실행 시 마다 (즉, 해당 계정을 새로 시작하는 경우에) 새 글의 시작 지점과 시간을 표시하는 기능이 추가되었습니다.

그런데 우측 그림을 자세히 보시면 좌측 그림과는 달리 새 글 표시 바로 아래에 Load More... 라는 셀이 있는 것을 보실 수 있습니다. 이는 새로 가져오는 트윗의 갯수가 최대 수치 (한 번에 가져오는 새 글 수는 최대 200개입니다. 이는 트위터 API가 실제로 한 번에 가져올 수 있는 최대 트윗 갯수입니다.) 를 초과할 경우 발생하게 됩니다. 즉, 지난 번 로그인 이후 새로 가져와야 할 트윗의 갯수가 200개를 넘을 경우, 계속해서 새 트윗과 기존 트윗 사이에 Load More... 셀을 배치하여 끊김없이 그 사이의 트윗들을 확인할 수 있도록 하였습니다. 그런데 의외로 이 기능이 구현되어 있지 않은 트위터 앱들을 종종 볼 수 있습니다. 그러한 앱들의 경우, 새로 가져온 트윗과 기존 트윗의 사이에 존재하는 트윗들을 확인할 방법이 없습니다.

7. 내 프로필 및 리스트 보기 기능이 추가되었습니다.

<그림 8. 내 프로필 보기>

이미지 하단 탭을 보시면 기존에 없던 More 탭이 새로 생긴 것을 확인할 수 있습니다. 해당 탭 진입 후 상단의 My Profile을 누르면 자신의 상세 정보를 바로 확인할 수 있습니다. 또한 그 아래에 있는 My Lists를 누르면 나의 리스트 정보를 바로 확인할 수 있습니다.

8. 사용자 이름으로 검색하는 기능 추가

<그림 9. 사용자 이름으로 검색>

More 탭 내에서 Search User를 선택하면 사용자 이름 검색이 가능합니다. 진입 시 해당 사용자가 팔로우하고 있는 사용자 목록을 트위터 서버로부터 받아와서 보여주게 됩니다.

9. 컬러 테마 설정 기능 위치 변경

<그림 10. 컬러 테마 설정>

More 탭 내에서 Color Theme를 선택하면 원하는 컬러 테마로 설정하는 것이 가능합니다. 지난 버전에서는 TwitStar 외부에서 (정확히는 홈 스크린의 설정 앱) 설정하는 방식이었으나, 찾기가 어렵다는 의견이 많아 TwitStar 내부로 위치를 변경하였습니다.

10. XAuth 인증 방식 사용

트위터 측에서 권장하는 OAuth / XAuth 인증 방식을 지원함에 따라, 트위터 서버와의 각종 통신 시 보다 안전한 기능 수행이 가능해졌습니다.

11. Twipl 서비스 연동

이미지 업로딩 서비스로 Twipl (http://www.twipl.net)을 지원합니다. 기존 버전에 비해 보다 안정적인 서비스 이용이 가능해졌습니다.

이상 간략하게 이번 버전 추가 사항들을 정리해 보았습니다. TwitStar 많이들 사랑해 주시고 개선이 필요한 부분은 언제든지 @iTwitStar로 알려주시기 바랍니다. 여러 분의 관심어린 조언 하나하나가 더 나은 TwitStar를 만들어 나가는데 큰 힘이 됩니다.

[Chromium] Patch (about Issue 19466) has been accepted.

Issue 19466 : Dropped-down navigation is listed far away from Arrow keys.

Description

Dropped-down navigation is listed far away from Arrow keys.

This is the patch related to the issue 6878

*** How to produce the bug ***

1. Execute browser with non-maximized.

2. Navigate to some (one or more) pages in the same tab. (To make the navigation history)

3. Move the browser to the left direction till the part of the left arrow button (back button) is put outside of the screen.

4. Click the right mouse button on the left arrow button.

5. The popup menu is displayed not on the click position but on the wrong position.

*** The reason of the bug ***

The popup menu is actually displayed by TrackPopupMenuEx() in the Menu::RunMenuAt(). TrackPopupMenuEx
receives the start position (x, y) by arguments.

In normal case (0 < x < screen_width, 0 < y < screen_height), it creates the popup menu on the specified position.
But when the x value is below zero (0 > x), it displays the popup menu on the wrong position.

This is why above bug is produced.

*** How to fix it ***

Check the x value before it is sent to the TrackPopupMenuEx(). When it has the value below the zero, modify the
value to the zero (0) so the minus value is not delivered to the function.

[Chromium] Patch (about Issue 18095) has been accepted.

Issue 18095 : In chromium, context menu is displayed when right mouse button is released.

Fixed at Revision 18170.

Description

In chromium, a context menu is displayed when right mouse button is released.
(e.g. context menu for a tab, etc)

However, unlike other cases, a context menu for an app (the context menu that is displayed on the title bar) is created when right mouse button is pressed.

So, for a consistency, modify it so that the context menu for an app is displayed not on mouse-press but on mouse-release like other cases.

* Above is also described on the Comment 11 by sy3620, Jan 08, 2009 in Issue5695. This is not about fixing the bug, but suggesting about changing the way the context menu for an app is displayed.
